ZABBIX NOTES
=============

OFFICIAL URL
-------------
https://www.zabbix.com

VENDOR
------
Zabbix LLC

TYPE
----
Enterprise monitoring and observability platform

MAIN PURPOSE
------------
- Monitor servers, networks, applications, and services
- Collect metrics and detect issues in real time
- Provide alerts and dashboards for infrastructure health

KEY FEATURES
------------
- Real-time monitoring
- Alerting and notifications (email, SMS, scripts)
- Custom dashboards and graphs
- Agent-based and agentless monitoring
- Auto-discovery of devices and services
- Event correlation and triggers

COMMON USE CASES
-----------------
- Server infrastructure monitoring (Linux, Windows, Unix)
- Network device monitoring (routers, switches)
- Application performance monitoring
- SLA and uptime tracking
- Capacity planning and reporting

COMPONENTS
-----------
- Zabbix Server (core engine)
- Zabbix Agent (installed on monitored hosts)
- Zabbix Proxy (distributed monitoring)
- Zabbix Frontend (web interface)

DATA COLLECTION METHODS
-----------------------
- Zabbix agent
- SNMP (network devices)
- IPMI (hardware monitoring)
- HTTP checks
- Custom scripts

BASIC IDEA
----------
Zabbix is a centralized monitoring system used to track infrastructure health, detect failures, and generate alerts for IT environments in real time.
