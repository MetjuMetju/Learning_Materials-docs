# Instance Configuration

#### System Properties
Global configuration settings that control how a ServiceNow instance behaves.

Examples:
- Email configuration
- User interface settings
- Security settings
- Platform behavior

#### Plugins
Packages that activate additional ServiceNow features and applications.

#### Applications
Applications provide functionality for specific business needs and can be installed, configured, or developed.

#### Personalization
User-specific changes that modify the appearance or experience without changing the system configuration.

Examples:
- List columns
- Favorites
- Filters

#### Customization
Changes made to adapt ServiceNow functionality to business requirements.

Examples:
- Custom fields
- Forms
- Workflows
- Business Rules

#### User Interface
The different ways users interact with ServiceNow.

Examples:
- Lists
- Forms
- Workspaces
- Portals

#### Instance Configuration
The process of configuring applications, settings, interfaces, and platform behavior according to organizational needs.
