# Configuring Applications for Collaboration

#### Lists
A collection of records displayed from a ServiceNow table.

Users can:
- Filter records
- Sort records
- Configure columns
- Export data

#### Filters
Conditions used to display only records matching specific criteria.

Example:
Show only active incidents assigned to a group.

#### Form Anatomy
The structure of a ServiceNow form.

Includes:
- Header
- Fields
- Sections
- Related Lists
- Form buttons

#### Form Configuration
Controls how forms appear and behave.

Examples:
- Add fields
- Remove fields
- Create sections
- Configure related lists

#### Form Templates
Predefined values that automatically populate fields when creating records.

#### Task Management
The Task table is a central ServiceNow table used for managing work records.

Examples:
- Incident
- Problem
- Change
- Request

#### Visual Task Boards
A Kanban-style interface for organizing and managing tasks visually.

#### Dashboards
A collection of reports and widgets showing information and metrics.

#### Platform Analytics
Tools used to analyze trends, performance, and operational data.

#### Notifications
Messages sent to users based on events or conditions.

Examples:
- Email notifications
- System notifications
- Alerts
